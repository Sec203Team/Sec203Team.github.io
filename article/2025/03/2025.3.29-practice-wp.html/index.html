<?php

highlight_file("/var/www/html/index.php");
error_reporting(0);

$a=$_GET['action'];

if($a=="getNumber"){
    if($m=$_GET['m']){
    getNumber($m);
    }else{
       echo "do you have m?";
    }
}else{
    die("looking the code,OK?");
}



function getArray($total, $times, $min, $max)
{
    $data = array();
    if ($min * $times > $total) {
        return array();
    }
    if ($max * $times < $total) {
        return array();
    }
    while ($times >= 1) {
        $times--;
        $kmix = max($min, $total - $times * $max);
        $kmax = min($max, $total - $times * $min);
        $kAvg = $total / ($times + 1);
        $kDis = min($kAvg - $kmix, $kmax - $kAvg);
        $r = ((float)(rand(1, 10000) / 10000) - 0.5) * $kDis * 2;
        $k = round($kAvg + $r);
        $total -= $k;
        $data[] = $k;
    }
    return $data;
}

function getNumber($m){
    $number = getArray(1000,20,10,999);
    if($number[$m]==$m ){
        echo shell_exec('cat /flag');
    }else{
        echo 'number is right?';
    }
}

looking the code,OK?

代码的结构大致如下：当通过GET请求传入参数action=getNumber时，会检查是否存在参数m。如果存在，就调用getNumber函数，并将m传递进去。否则提示“do you have m?”。其他情况下显示“looking the code,OK?”。

接下来看getNumber函数。它调用了getArray(1000,20,10,999)，生成一个包含20个元素的数组。然后检查数组的第m个元素，是否等于m个元素是否等于m本身。如果满足，就输出flag；否则显示错误。

1.参数传递
?action=getNumber&m=目标值。
若m不存在或action不等于getNumber，代码会直接报错。

2.*数组生成规则
getArray(1000, 20, 10, 999)将*1000拆分为20****个整数，每个数在10~999之间。

￮ 每个数的生成依赖随机性（rand(1,10000)/10000），但需满足总和为 1000。

￮ 数组索引范围为0~19，数值下限为 10，因此仅当**m在 10~19 **时才可能满足$number[$m] == $m。

**3.**触发条件漏洞
若数组的第m个元素值恰好等于m，则输出 flag。例如：

￮ 当m=15时，需$number[15] = 15。

攻击方法

**1.**暴力穷举法
由于数组生成存在随机性，可通过多次请求m=10至m=19尝试命中条件。

payload:?action=getNumber&m=10

2.脚本
编写脚本遍历m=10~19并发起请求，直至返回 flag：

python
import requests 

for m in range(10, 20):
    url = f"http://target.com/index.php?action=getNumber&m={m}" 
    response = requests.get(url) 
    if "flag{" in response.text: 
        print(f"Success! Flag found at m={m}: {response.text}") 
        break

找到传参点

原来的序列化是O:1:”a”:2:{…}，表示有两个属性

代码里有一个类a，里面有uname和password两个公共属性。构造函数初始化这两个属性，__wakeup魔术方法在反序列化时被调用。如果password等于字符串’203’，就会包含flag.php 并输出flag，否则输出错误信息。

然后有一个filter函数，用str_replace把字符串里的’win’替换成’wrong’。接下来，用户输入的203参数赋值给uname，password设为1，序列化一个a类的实例，经过filter处理后输出，再反序列化触发__wakeup。

字符串可逃逸的基础**:**反序列化分隔符
反序列化以;}结束，后面的字符串不影响正常的反序列化

序列化后的结果经过filter替换会把每个win替换成wrong，所以每个win变成wrong，uname长度增加（win是3字符，wrong是5，所以每次替换增加2字符）。只要输入的win次数足够多，导致替换后的uname部分的字符串长度超过原来的计算，从而使得后面的”;s:8:”password”;s:3:”203”;}部分被正确解析，覆盖原来的password为203。

easyrsa

RSA 加密的过程是将明文转换为数字 m，然后用公钥 (e, n) 计算 c = m^e mod n。解密则是用私钥 d 计算 m = c^d mod n。

from Crypto.Util.number import *
from flag import flag


def nextPrime(n):
    n += 2 if n & 1 else 1
    while not isPrime(n):
        n += 2
    return n

p = getPrime(1024)
q = nextPrime(p)
n = p * q
e = 0x10001
d = inverse(e, (p-1) * (q-1))
c = pow(bytes_to_long(flag.encode()), e, n)
